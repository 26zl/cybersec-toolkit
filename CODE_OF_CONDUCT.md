# Code of Conduct

This project is for educational, defensive, and authorized security work. Treat contributors, maintainers, and issue reporters with respect, especially when discussing offensive security topics.

## Expected behavior

- Be constructive, patient, and specific.
- Assume good intent, but verify claims with evidence.
- Keep issues and pull requests focused on the project.
- Redact secrets, private targets, customer data, and personal information.
- Discuss dual-use tooling responsibly and within authorized scope.

## Unacceptable behavior

- Harassment, threats, hate speech, or personal attacks.
- Publishing private vulnerability details outside the process in `SECURITY.md`.
- Sharing unauthorized targets, stolen data, credentials, or exploit output.
- Repeated spam, low-effort noise, or intentionally misleading reports.

## Enforcement

Maintainers may edit or remove content, close issues or pull requests, and block accounts that violate this code of conduct. Security vulnerabilities should be reported privately through GitHub Security Advisories, not public issues.
